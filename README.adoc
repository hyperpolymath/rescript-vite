= rescript-vite
:toc: macro
:toclevels: 3

The definitive Vite plugin for ReScript -- zero-config, all known pain points
solved. HMR, compiler integration, error overlay, PascalCase resolution,
auto optimizeDeps, rewatch support, rescript.json auto-detection, and optional
BoJ build orchestration.

toc::[]

== What is rescript-vite?

rescript-vite integrates the ReScript compiler into Vite's build pipeline
with *zero configuration*. It reads your `rescript.json`, auto-detects your
suffix and module format, excludes `@rescript/*` packages from pre-bundling,
resolves PascalCase modules on case-sensitive filesystems, ignores build
artifacts in the watcher, spawns the compiler in the right mode, and forwards
diagnostics to Vite's error overlay. It Just Works.

=== Features

* *Zero config* -- `plugins: [rescriptPlugin()]` and you're done. Everything
  is auto-detected from `rescript.json`.
* *PascalCase module resolution* -- fixes the #1 Linux pain point where
  ReScript emits lowercase imports but actual files are PascalCase on disk.
* *Auto `optimizeDeps` exclusion* -- `@rescript/core`, `@rescript/runtime`,
  `@rescript/react`, and `rescript` are excluded from Vite's dependency
  pre-bundling, which breaks on their subpath exports.
* *Build artifact watcher ignore* -- `.ast`, `.cmj`, `.cmi`, `.cmt`, and
  `lib/` directories are ignored to prevent spurious reload loops.
* *rescript.json auto-detection* -- reads your suffix (`.res.js` vs `.res.mjs`),
  module format (`esmodule` vs `commonjs`), and `in-source` setting automatically.
* *Rewatch support* -- auto-detects `rewatch` binary (ReScript 12+) and uses
  native `rewatch watch`/`rewatch build` commands.
* *Automatic compiler management* -- spawns `rescript build` (one-shot) or
  `rescript build -w` / `rewatch watch` depending on whether Vite is in
  `build` or `serve` mode.
* *HMR for .res files* -- detects recompiled output files and triggers
  hot module replacement in the browser.
* *Error overlay integration* -- pushes ReScript compiler errors to Vite's
  built-in error overlay with file, line, and column info.
* *ANSI color forwarding* -- sets `NINJA_ANSI_FORCED=1` for colored compiler
  output in the terminal.
* *In-source: false support* -- correctly maps `.res` files to compiled output
  in `lib/es6/` or `lib/js/` for out-of-source compilation.
* *Deno compatible* -- auto-detects the Deno runtime and adjusts compiler
  invocation accordingly.
* *BoJ build orchestration* -- optionally delegates builds to a running
  BoJ ssg-mcp cartridge for incremental builds, caching, and telemetry.
* *All features toggleable* -- `autoOptimizeDeps`, `autoResolve`,
  `autoIgnoreArtifacts` can be individually disabled if needed.
* *panic-attacker compatible* -- diagnostic format maps cleanly to SARIF-style
  output used by panic-attacker weak-point scanning.

== Installation

=== Using Deno

[source,bash]
----
deno add npm:rescript-vite
----

=== Using npm (where Deno is not available)

[source,bash]
----
npm install rescript-vite
----

=== Peer dependencies

rescript-vite expects these as peer dependencies:

* `rescript` ^12.0.0
* `@rescript/core` ^1.0.0
* `vite` ^6.0.0

== Usage

=== Basic setup

[source,javascript]
----
// vite.config.js
import rescriptPlugin from "rescript-vite";

export default {
  plugins: [rescriptPlugin()],
};
----

=== With options

[source,javascript]
----
// vite.config.js
import { make } from "rescript-vite";

export default {
  plugins: [
    make({
      useDeno: true,
      logLevel: "verbose",
      compilerFlags: ["-warn-error", "+a"],
    }),
  ],
};
----

=== With BoJ build orchestration

[source,javascript]
----
// vite.config.js
import { make } from "rescript-vite";

export default {
  plugins: [
    make({
      boj: true,
      bojEndpoint: "http://localhost:7077/mcp/ssg",
      logLevel: "info",
    }),
  ],
};
----

When `boj: true` is set, the plugin probes the BoJ ssg-mcp endpoint at
build start. If the server is reachable, production builds are delegated to
it. If unreachable, the plugin falls back to direct compiler invocation.

== API Reference

=== `make(options?)`

Creates a Vite plugin instance. Returns a standard Vite plugin object with
`name: "rescript-vite"` and `enforce: "pre"`.

==== Options

[cols="1,1,2,1"]
|===
|Option |Type |Description |Default

|`boj`
|`boolean`
|Enable BoJ ssg-mcp build orchestration. The plugin probes the endpoint
 at build start and falls back to direct compilation if unavailable.
|`false`

|`bojEndpoint`
|`string`
|URL of the BoJ ssg-mcp cartridge endpoint.
|`http://localhost:7077/mcp/ssg`

|`useDeno`
|`boolean`
|Use Deno to run the ReScript compiler (`deno run -A npm:rescript`).
|Auto-detected

|`rescriptBin`
|`string`
|Path to the rescript binary. When set, the plugin invokes this binary
 directly instead of using `npx`.
|`undefined` (uses `npx`)

|`compilerFlags`
|`string[]`
|Extra flags passed to the ReScript compiler.
|`[]`

|`logLevel`
|`"silent" \| "info" \| "verbose"`
|Controls console output verbosity.
|`"info"`

|`suffix`
|`string`
|Output suffix override (e.g., `".res.mjs"`). Auto-detected from `rescript.json`.
|Auto-detected

|`useRewatch`
|`boolean`
|Use `rewatch` instead of `rescript build -w` for watch mode.
|Auto-detected (checks for `rewatch` binary)

|`autoOptimizeDeps`
|`boolean`
|Auto-exclude `@rescript/*` packages from Vite's dependency pre-bundling.
|`true`

|`autoResolve`
|`boolean`
|Auto-resolve PascalCase module imports on case-sensitive filesystems (Linux).
|`true`

|`autoIgnoreArtifacts`
|`boolean`
|Auto-ignore ReScript build artifacts (`.ast`, `.cmj`, `.cmi`, `.cmt`, `lib/`) in Vite's file watcher.
|`true`
|===

=== Default export

The module's default export is a plugin instance created with default options:

[source,javascript]
----
import rescriptPlugin from "rescript-vite";
// Equivalent to: make()
----

=== `rescript-vite/config` (RescriptConfig)

The config reader is available as a separate export for direct use:

[source,javascript]
----
import { read, getOutputPath, defaultRescriptConfig } from "rescript-vite/config";

const config = read(".");
console.log(config.suffix);       // ".res.js" or ".res.mjs"
console.log(config.packageSpec);  // { moduleFormat: "esmodule", inSource: true }
----

==== RescriptConfig API

* `read(projectRoot)` -- Read `rescript.json` or `bsconfig.json` from a directory. Returns parsed config or defaults.
* `getOutputPath(config, resFile)` -- Get the compiled output path for a `.res` file (handles in-source and out-of-source).
* `defaultRescriptConfig` -- Default configuration when no `rescript.json` is found.

=== `rescript-vite/boj` (BojBridge)

The BoJ bridge is available as a separate export for direct use:

[source,javascript]
----
import { make, probe, requestBuild, recentEvents, disconnect } from "rescript-vite/boj";

const bridge = make("http://localhost:7077/mcp/ssg");
const connected = await probe(bridge);
----

==== BojBridge API

* `make(endpoint?)` -- Create a bridge instance (disconnected by default).
* `probe(bridge)` -- Attempt to connect; returns `true` on success.
* `isConnected(bridge)` -- Check connection state.
* `requestBuild(bridge, request)` -- Send a build request via JSON-RPC.
* `recentEvents(bridge, limit?)` -- Get recent telemetry events (default limit: 20).
* `disconnect(bridge)` -- Disconnect the bridge.

== BoJ Integration

The plugin integrates with the *BoJ (Bundle of Joy)* server's `ssg-mcp`
cartridge. BoJ is a multi-cartridge build orchestration system; the ssg-mcp
cartridge handles static site generation and ReScript compilation with:

* *Build caching* -- avoids redundant recompilation when source files
  have not changed.
* *Telemetry* -- records build events (duration, file count, cache hit rate,
  diagnostic count) for monitoring and analysis.
* *MCP protocol* -- communicates via JSON-RPC 2.0 over HTTP, calling the
  `tools/call` method with the `ssg_build` tool name.

When `boj: true` is set:

1. On `buildStart`, the plugin probes `{bojEndpoint}/health`.
2. If healthy, production builds are sent as `ssg_build` requests.
3. If the BoJ request fails, the plugin falls back to direct `rescript build`.
4. Telemetry events are recorded in the bridge for later inspection.
